# 多币种处理

本应用可以在币种设置中启用多种币种，并在它们之间切换。币种记录在每条分录明细上，因此可以按币种分别管理外币和加密资产余额。

## 核心思路

画面上方的币种切换器决定输入和显示所使用的币种。例如，选择USD后在美元银行账户中记录1,000的存款，该分录将以USD保存。

账户并不限定为单一币种。您既可以在同一账户中保留多种币种余额，也可以按币种和用途创建不同账户。需要明确分开余额时，可以使用“USD银行账户”“BTC－日常”等专用账户。

## 开始使用外币

1. 在币种设置中启用USD、EUR等目标币种。
2. 通过画面上方的币种切换器选择该币种。
3. 如果希望按用途分开余额，请创建专用账户。
4. 使用期初余额表单，以当前选择的币种记录余额。

## 记录外币交易

外币存取款与普通条目输入方式相同。以外币输入金额。

| 交易示例 | 输入方法 |
| --- | --- |
| 向美元账户存入1,000美元 | 选择美元账户作为收入目标，输入1,000 |
| 从美元账户支付100美元支出 | 选择美元账户作为支出来源，输入100 |
| 将美元账户转换为日元账户 | 在两个账户之间使用多行分录 |

## 币种换算分录

在日元账户和外币账户之间进行换算时，使用多行分录。

**示例：将1,000美元（按150日元/美元购入）兑换为145,000日元**

| 借方 | 贷方 |
| --- | --- |
| 银行账户（日元）145,000 | 美元账户 1,000（成本基础等值150,000日元） |
| 汇兑损失 5,000 | |

购入汇率与兑换汇率之间的差额成为汇兑损益，记录为收入或支出。

本应用支持允许借方/贷方金额不匹配的币种兑换分录（`is_currency_exchange`）。在多行分录表单中，将分录记录为币种兑换可以保存不同币种金额的组合。

## 在财务报表中的显示

财务报表通常只显示币种切换器当前所选币种的余额。启用“包含所有币种”后，其他币种余额会按照币种设置中的汇率换算为显示币种，并计入合计。

换算值会随价格来源和获取时间变化。账簿仍保留原币种数量；显示换算不会修改分录。

## 添加或删除币种

添加币种不会影响现有账户或分录。仍有余额的币种无法停用；不再需要时，请先通过转账或币种兑换分录将余额归零。

## 加密资产与币种

加密资产支持两种使用模式。

- 用于日常支付时，请在币种设置中启用BTC等加密币种，通过币种切换器选择后记录日常收入、支出和转账。如需分开管理余额，可创建“BTC－日常”等现金账户。
- 用于投资或投机时，请在加密资产类别中创建“BTC－投资”等账户，将买入、卖出、奖励和手续费记录为投资活动。

同一BTC可以同时用于两种目的。即使币种代码相同，不同账户的余额也会保持独立，余额核对也会逐个账户比较。投资账户可以获取钱包实际余额；包括日常使用账户在内的所有账户都可以手动输入实际余额。详情请参阅加密资产余额核对指南。
