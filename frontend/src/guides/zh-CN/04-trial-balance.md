# 试算表与信用卡对账

试算表页面检查账面余额是否与实际余额一致。从财务报表 -> 试算表打开。

## 功能

| 标签页 | 用途 |
| --- | --- |
| 实际输入 | 输入实际账户余额或信用卡使用金额 |
| 差错验证 | 比较实际值与账面余额 |
| 不明资金 | 临时吸收无法解释的差额 |
| 预算一致性检查 | 检查分录与预算之间的不匹配及预算布局 |

## 账户和信用卡

保存银行账户、现金、负债或信用卡使用的实际余额后，差错验证会显示账面值、实际值和差额。如果存在差额，检查相同金额的候选项目、近期分录、错误日期、重复记录、退款以及方向错误的转账。

信用卡常出现差额，因为使用日、账单截止日、确认日、扣款日和还款月是分开的。当账单确认后，保存相关还款月的金额并与账簿使用情况进行比较。

## 预算一致性检查

预算一致性检查验证分录的收入/支出金额是否与已保存的预算分配金额匹配。它检测缺少分配或金额不匹配的分录。

对于多行分录和预算跟踪之外的交易，除非保存了明确的分配，否则应用不会猜测预算类别。如有需要，从账簿编辑分录，或使用预算调整仅修正当期预算。

此标签页还显示预算布局表。它将预算类别上配置的目标持有账户与实际现金/银行余额进行比较，然后显示关于在账户组之间转移多少资金或增加/减少多少预算的提示。
