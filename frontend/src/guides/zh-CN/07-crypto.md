# 加密资产监控

加密资产监控将钱包地址链接到加密资产账户，以便您查看余额和日元等值价值。从财务报表 -> 加密资产进行管理。

除了根据正常分录计算的余额外，应用还可以获取实际的链上余额，并将其反映在财务报表中显示的加密资产估值中。

## 主要支持类型

| 类型 | 内容 |
| --- | --- |
| BTC | 比特币地址余额 |
| ETH | 以太坊地址余额 |
| SOL | Solana钱包的SOL余额 |
| SKR | Solana系列地址上的SKR余额 |
| mSOL | 通过Marinade质押的SOL等值余额 |
| SOL Stake | Solana原生质押账户余额 |

币安集成目前已禁用。如果您想管理交易所余额，请创建加密资产账户并根据需要通过手动分录或实际余额检查进行管理。

## 添加钱包

1. 在设置中，创建一个加密类别的资产账户。
2. 打开财务报表 -> 加密资产。
3. 添加钱包，然后选择链、地址和关联账户。
4. 使用余额获取检查可用数量和日元等值金额。
5. 保存后钱包将显示在加密资产列表中。

通常一个账户关联一个钱包设置。即使使用同一个Solana地址，您可能希望分别处理SOL、SKR、mSOL等资产，因此需要时请创建单独的账户。

## 日元价值的处理方式

在加密资产页面上，应用将获取的数量乘以价格数据来显示日元等值价值。价格从外部价格源获取，可以在屏幕上刷新。

对于加密类别的账户，财务报表中显示的估值可能会被从获取数量和价格计算的价值覆盖，而不是仅从分录计算出的余额。这使您可以查看更接近当前市场价值的总资产。

## 与分录的关系

加密资产监控用于检查持有数量和估值。它不会自动为每笔购买、销售、兑换、转账、手续费、已实现损益创建分录。

当您购买加密资产时，仍需要一笔减少支付资产并增加加密资产账户的分录。当您出售时，记录现金/存款的增加、加密资产的减少，以及必要的损益。

## Solana系列资产注意事项

SOL、SKR、mSOL和SOL Stake可能使用相似的地址格式。自动检测可能无法正确区分它们，因此需要时请明确选择链。

Marinade mSOL和原生质押账户使用与普通钱包余额不同的余额获取方法。如果余额获取失败，请检查地址类型是否正确，以及目标是钱包地址还是质押账户地址。

## 管理股票和有价证券

股票、共同基金等有价证券通过创建投资类别的资产账户进行管理。余额根据分录计算。

### 购买分录

购买股票时使用多行分录。

| 借方 | 贷方 |
| --- | --- |
| 投资账户（持有股份） | 支付账户（现金/存款） |
| 买入手续费（支出，如适用） | |

在备注字段中记录买入价格或成本基础，将有助于日后计算损益。

### 出售分录

如果出售价格超过成本基础，产生已实现收益；如果低于成本基础，产生已实现损失。

**有收益的出售：**

| 借方 | 贷方 |
| --- | --- |
| 收款账户（现金/存款） | 投资账户（持有股份） |
| | 有价证券出售收益 |

**有损失的出售：**

| 借方 | 贷方 |
| --- | --- |
| 收款账户（现金/存款） | 投资账户（持有股份） |
| 有价证券出售损失 | |

"有价证券出售收益"和"有价证券出售损失"可作为系统账户使用。

## 记录加密资产交易

加密资产出售遵循与有价证券相同的逻辑。

| 类型 | 使用的系统账户 |
| --- | --- |
| 出售收益 | 加密资产出售收益 |
| 出售损失 | 加密资产出售损失 |

加密资产监控用于检查持有量和估值。它不会自动记录购买、销售、兑换或转账。请以多行分录形式单独记录实际交易。

### 估值与账面价值的差异

加密资产监控启用后，财务报表以当前市场价格显示估值。然而，账簿余额基于获取时的入账金额。这两者之间的差距是未实现损益，在您出售之前不会反映在账簿中。

对于股票，也没有自动的月度重估。成本基础保留在账簿中，实际损益在出售时记录。

## 家庭理财说明

加密市场价值可能剧烈波动。概览和财务报表中显示的估值是用于了解当前家庭净资产的估算值。它不能完全替代税务成本基础、损益计算、移动平均法、总平均法或手续费处理。

在大额交易之后，请单独核对付交易所和钱包历史记录的分录。
