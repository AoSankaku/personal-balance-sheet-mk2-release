# 加密资产余额核对

余额核对将钱包地址链接到加密资产账户，以便比较实际余额和账簿余额。从财务报表 -> 余额核对进行管理。

仅从链上获取用于余额核对的持有数量。不获取市场价值或未实现损益，也不会覆盖财务报表中的账簿余额。

## 加密资产的两种使用模式

账户的加密资产类别与币种承担不同的作用。类别表示持有该资产的目的，币种表示分录使用的记账单位。

| 使用模式 | 设置示例 | 主要用途 |
| --- | --- | --- |
| 投资或投机持有 | 在加密资产类别中创建“BTC－投资”账户 | 不用于日常消费，将买入、卖出、转账、奖励和手续费作为投资活动记录。可以关联钱包以获取实际余额 |
| 日常支付币种 | 在币种设置中启用BTC，并根据需要创建“BTC－日常”等现金账户 | 通过币种切换器选择BTC，以BTC记录日常收入、支出和转账 |

两种模式可以同时使用。例如，将“BTC－日常”和“BTC－投资”设为不同账户，即可按用途分别管理同一BTC币种的余额。余额核对会按账户比较，而不会仅根据币种代码把它们合并。

只有在页眉中选择BTC等对应加密币种时，才会通过明确操作为加密资产类别账户获取钱包余额。普通页面刷新不会获取余额。包括日常使用账户在内，每个账户的实际余额仍可手动输入；获取结果不会覆盖您手动编辑过的值。

## 主要支持类型

| 类型 | 内容 |
| --- | --- |
| BTC | 比特币地址余额 |
| ETH | 以太坊地址余额 |
| SOL | Solana钱包的SOL余额 |
| SKR | Solana系列地址上的SKR余额 |
| mSOL | 通过Marinade质押的SOL等值余额 |
| SOL Stake | Solana原生质押账户余额 |

币安集成目前已禁用。如果您想管理交易所余额，请创建加密资产账户并根据需要通过手动分录或实际余额检查进行管理。

## 添加钱包

1. 在设置中，创建一个加密类别的资产账户。
2. 在页眉中选择目标加密币种，打开财务报表 -> 余额核对，然后选择实际余额输入。
3. 添加钱包，然后选择链、地址和关联账户。
4. 使用“获取并应用”将可用数量填入实际余额。
5. 保存快照并检查它与账簿余额之间的差异。

通常一个账户关联一个钱包设置。即使使用同一个Solana地址，您可能希望分别处理SOL、SKR、mSOL等资产，因此需要时请创建单独的账户。

## 货币的处理方式

财务报表使用可用价格换算账簿余额，并以显示货币展示等值金额。价格来自外部价格源，可以刷新。

从钱包获取的数量保留其原始货币，仅作为余额核对中的实际余额；它不会替代分录计算出的金额。

## 与分录的关系

余额核对用于比较持有数量和账簿记录。它不会自动为每笔购买、销售、兑换、转账、手续费、已实现损益创建分录。

当您购买加密资产时，仍需要一笔减少支付资产并增加加密资产账户的分录。当您出售时，记录现金/存款的增加、加密资产的减少，以及必要的损益。

## Solana系列资产注意事项

SOL、SKR、mSOL和SOL Stake可能使用相似的地址格式。自动检测可能无法正确区分它们，因此需要时请明确选择链。

Marinade mSOL和原生质押账户使用与普通钱包余额不同的余额获取方法。如果余额获取失败，请检查地址类型是否正确，以及目标是钱包地址还是质押账户地址。

## 管理股票和有价证券

股票、共同基金等有价证券通过创建投资类别的资产账户进行管理。余额根据分录计算。

### 购买分录

购买股票时使用多行分录。

| 借方 | 贷方 |
| --- | --- |
| 投资账户（持有股份） | 支付账户（现金/存款） |
| 买入手续费（支出，如适用） | |

在备注字段中记录买入价格或成本基础，将有助于日后计算损益。

### 出售分录

如果出售价格超过成本基础，产生已实现收益；如果低于成本基础，产生已实现损失。

**有收益的出售：**

| 借方 | 贷方 |
| --- | --- |
| 收款账户（现金/存款） | 投资账户（持有股份） |
| | 有价证券出售收益 |

**有损失的出售：**

| 借方 | 贷方 |
| --- | --- |
| 收款账户（现金/存款） | 投资账户（持有股份） |
| 有价证券出售损失 | |

"有价证券出售收益"和"有价证券出售损失"可作为系统账户使用。

## 记录加密资产交易

加密资产出售遵循与有价证券相同的逻辑。

| 类型 | 使用的系统账户 |
| --- | --- |
| 出售收益 | 加密资产出售收益 |
| 出售损失 | 加密资产出售损失 |

余额核对用于比较持有量和账簿记录。它不会自动记录购买、销售、兑换或转账。请以多行分录形式单独记录实际交易。

### 实际余额与账簿余额的差异

核对画面分别显示获取的实际余额和根据分录计算的账簿余额。如有差异，请检查钱包历史并补记缺少的分录；获取余额本身不会修改账簿。

对于股票，也没有自动的月度重估。成本基础保留在账簿中，实际损益在出售时记录。

## 家庭理财说明

加密资产的市场价格可能剧烈波动。概览和财务报表中显示的换算值仅用于估算当前家庭净资产，不能替代税务成本基础、损益计算、移动平均法、总平均法或手续费处理。

在大额交易之后，请单独核对付交易所和钱包历史记录的分录。
