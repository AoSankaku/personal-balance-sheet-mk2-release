# 预算系统（虚拟桶）

预算类别是叠加在实际现金和银行账户之上的虚拟桶。实际账户余额保持不变，而预算系统按用途分配这些资金，例如自由支出、必要支出、旅行基金或投资储备。

## 核心概念

| 项目 | 作用 |
| --- | --- |
| 账户 | 实际账簿项目，如银行账户、信用卡、工资或餐饮支出 |
| 预算类别 | 基于用途的现金/存款虚拟桶 |
| 关联支出账户 | 决定哪些支出分录消耗预算 |
| 目标持有账户 | 描述您打算将预算资金存放在何处 |

从设置 -> 预算设置创建类别。一个类别可以有分组、目标余额、余额上限、溢出目的地、关联支出账户和目标持有账户。

## 滚动、重置和上限

常规预算全额滚动，无论是正数还是负数。负预算表示超支，因此会减少可分配资金而非增加。

如果历史导入或修正使余额变得不现实，请使用输入 -> 预算调整 -> 预算重置将该类别归零。概览页面会在当月显示一个带日期的重置徽章。余额上限可以将多余预算发送到另一个类别。

## 收入分配和调整

收入条目可以应用预算过滤器，将存入资金分配到各个类别。过滤器使用固定金额、上限和比例分配步骤。后续的手动预算调整可以增加或减少单个类别，并可以包含可选备注。

## 预算布局

现金/银行账户有一个"包含在可分配预算来源中"的设置。只有启用的账户才会计入可分配现金和预算布局中的实际余额。

预算布局表出现在设置 -> 预算设置和财务报表 -> 试算表 -> 预算一致性检查中。

| 显示 | 含义 |
| --- | --- |
| 目标 | 账户组预期的预算余额 |
| 实际 | 目标账户的实际现金/银行余额 |
| 差额 | 实际 - 目标 |

如果一个预算关联多个账户，或多个预算关联一个账户，相关的预算和账户会合并成一个组。存在差额时，提示会显示需要在账户组之间转移多少资金，或需要增加或减少多少预算。

## 转账和预算移动

在简易输入中输入账户转账时，您可以选择同时移动预算。选择源和目标预算类别记录预算转移，或将目标留空以消耗/消失预算，例如将预留资金用于投资时。

实际转账和预算移动是独立的事件。预算-账户关联是布局指导；它们不会强制收入分配和银行转账同时发生。

## 预算一致性

试算表 -> 预算一致性检查检测分录金额与已保存的预算分配金额之间的不匹配。对于多行分录和预算跟踪之外的交易，除非保存了明确的分配，否则应用不会猜测预算类别。
