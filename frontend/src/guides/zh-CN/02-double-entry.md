# 复式记账与简易输入

本应用中的所有账簿数据都以复式分录形式存储。在复式记账中，每笔交易同时记录"什么增加了"和"什么减少了"。这使得日后可以验证资产、负债、净资产、收入与支出之间的关系。

日常输入时您无需考虑借贷方向。在简易输入中，您选择交易类型、金额和对方账户，应用会自动创建匹配的分录。

## 简易输入创建的分录

| 输入类型 | 示例 | 应用内部操作 |
| --- | --- | --- |
| 支出 | 从银行账户支付餐饮费 | 增加支出，减少支付资产 |
| 收入 | 工资存入银行账户 | 增加资产，增加收入 |
| 转账 | 将资金从储蓄转到证券账户 | 减少一项资产，增加另一项资产 |
| 借入 | 朋友代您支付 | 增加支出或资产，增加负债 |
| 还款 | 偿还借款 | 减少负债，减少支付资产 |
| 借出 | 借钱给他人 | 增加借出资产，减少现金/存款 |
| 收回 | 收回借出的钱 | 增加现金/存款，减少借出资产 |

## 为何无需了解复式记账也能使用

简易输入询问的是现实生活中的信息。对于支出，您选择花钱的项目和付款来源。对于收入，您选择收入类型和存入账户。应用会根据账户类型自动确定借贷方向。

在账簿页面中，同一笔分录可以以简易视图和复式视图两种格式查看。如果不熟悉簿记，查看简易视图就足够了，只有在余额异常或需要处理特殊交易时才打开复式视图。

## 何时使用多行分录

当一笔交易涉及三个或更多账户时，应使用多行分录。例如：扣除手续费后的存款、混合了业务垫付和个人支出的交易，或者需要明确记录损益的销售。

对于多行分录，借方总额和贷方总额必须相等。无法保存不平衡的交易，这是保持账簿一致的核心规则。

## 借贷规则

在复式记账中，每种账户类型在一个方向上增加，在另一个方向上减少。

| 账户类型 | 增加方 | 减少方 |
| --- | --- | --- |
| 资产 | 借方 | 贷方 |
| 负债 | 贷方 | 借方 |
| 净资产 | 贷方 | 借方 |
| 收入 | 贷方 | 借方 |
| 支出 | 借方 | 贷方 |

了解此表后，您可以自行确定分录方向。例如，"从银行账户支付餐饮费"意味着银行账户是资产，因此记在贷方。餐饮支出增加，因此记在借方。这两行共同构成一个完整的分录。

## 如何阅读分录

在账簿页面的复式视图中，每一行显示为借方或贷方的明细。

| 侧 | 含义 |
| --- | --- |
| 借方（左侧） | 账户增加（资产、支出）或减少（负债、净资产、收入） |
| 贷方（右侧） | 账户增加（负债、净资产、收入）或减少（资产、支出） |

在一笔分录中，借方总额必须始终等于贷方总额。不匹配的分录无法保存。

## 账户类型及其作用

| 类型 | 示例 | 说明 |
| --- | --- | --- |
| 资产 | 银行账户、现金、借出款、加密资产 | 您拥有的资产 |
| 负债 | 信用卡、贷款、借款 | 您欠他人的金额 |
| 净资产 | 期初余额 | 净资产（资产减去负债） |
| 收入 | 工资、利息收入、销售收入 | 资金增加的原因 |
| 支出 | 餐饮、水电费、折旧 | 资金减少的原因 |

净资产计算为资产减去负债，因此很少直接输入。期初余额输入是其主要用途。

## 编辑注意事项

您可以从账簿页面编辑分录。通过简易输入创建的分录尽可能转换回原始输入格式。月度折旧分录和包含特殊系统账户的分录如果直接编辑，可能会破坏相关计算。在这些情况下，更安全的做法是编辑原始输入或设置。

当余额不匹配时，在凭猜测删除分录之前，先在试算表页面将差额与实际余额进行比较。找出原因后再修复分录，更容易理解对预算和报告的影响。
