# 複式簿記與簡易輸入

此應用程式中所有帳簿資料均以複式分錄儲存。在複式簿記中，每筆交易同時記錄「什麼增加了」和「什麼減少了」。這使得日後可以驗證資產、負債、淨值、收入與支出之間的關係。

日常輸入時您無需思考借貸問題。在簡易輸入中，您選擇交易型別、金額與對方帳戶，應用程式會自動建立對應的分錄。

## 簡易輸入建立的分錄

| 輸入型別 | 範例 | 應用程式內部動作 |
| --- | --- | --- |
| 支出 | 從銀行帳戶支付餐飲費用 | 增加支出並減少付款資產 |
| 收入 | 薪資存入銀行帳戶 | 增加資產並增加收入 |
| 轉帳 | 從儲蓄帳戶轉至證券帳戶 | 減少一項資產並增加另一項資產 |
| 借入 | 朋友代您付款 | 增加支出或資產，並增加負債 |
| 還款 | 償還借款 | 減少負債並減少付款資產 |
| 貸出 | 借錢給他人 | 增加貸出資產並減少現金／存款 |
| 收回 | 收回借出的款項 | 增加現金／存款並減少貸出資產 |

## 為何無需懂複式簿記也能使用

簡易輸入詢問的是現實生活中的資訊。對於支出，您選擇花費的項目與付款來源。對於收入，您選擇收入的型別與存入帳戶。應用程式會根據帳戶型別判斷借貸方向。

在帳簿頁面上，同一筆分錄可以簡易與複式兩種格式檢視。如果您不熟悉簿記，檢視簡易檢視就已足夠，僅在餘額異常或需要處理特殊交易時才開啟複式檢視。

## 何時使用多行分錄

當一筆交易涉及三個或更多帳戶時，請使用多行分錄。例如：有手續費扣除的存款、混合了公司墊款與個人支出的交易，或您想明確記錄損益的銷售。

多行分錄的借方總額與貸方總額必須相等。無法儲存不平衡的交易。這是保持帳簿一致性的核心規則。

## 借貸規則

在複式簿記中，每個帳戶型別在一方增加，在另一方減少。

| 帳戶型別 | 增加方 | 減少方 |
| --- | --- | --- |
| 資產 | 借方 | 貸方 |
| 負債 | 貸方 | 借方 |
| 權益（淨值） | 貸方 | 借方 |
| 收入 | 貸方 | 借方 |
| 支出 | 借方 | 貸方 |

瞭解此表格後，您就可以自行判斷分錄方向。例如，「從銀行帳戶支付餐飲費用」表示銀行帳戶是資產，應記在貸方；餐飲支出增加，應記在借方。這兩行共同構成一筆完整的日記分錄。

## 如何閱讀分錄

在帳簿頁面上，複式檢視將每一行顯示為借方或貸方的明細。

| 方 | 含義 |
| --- | --- |
| 借方（左側） | 帳戶增加（資產、支出）或減少（負債、權益、收入） |
| 貸方（右側） | 帳戶增加（負債、權益、收入）或減少（資產、支出） |

在一筆分錄中，借方總額必須始終等於貸方總額。不符的分錄無法儲存。

## 帳戶型別及其角色

| 型別 | 範例 | 說明 |
| --- | --- | --- |
| 資產 | 銀行帳戶、現金、貸出、加密貨幣 | 您擁有的東西 |
| 負債 | 信用卡、貸款、借入 | 您欠他人的金額 |
| 權益 | 期初餘額 | 淨值（資產減負債） |
| 收入 | 薪資、利息收入、出售收益 | 資金增加的原因 |
| 支出 | 餐飲、水電費、折舊 | 資金減少的原因 |

權益計算為資產減負債，因此很少直接輸入。期初餘額輸入是主要用途。

## 編輯注意事項

您可以從帳簿頁面編輯分錄。透過簡易輸入建立的分錄在可能的情況下可以轉換回原始輸入格式。每月折舊分錄和包含特殊系統帳戶的分錄若直接編輯可能會破壞相關計算。在這些情況下，建議改為編輯原始輸入或設定。

當餘額不符時，先在試算表頁面上將差額與實際餘額核對，再基於猜測刪除分錄。確定原因後再修正分錄，更容易理解對預算和報表的影響。
