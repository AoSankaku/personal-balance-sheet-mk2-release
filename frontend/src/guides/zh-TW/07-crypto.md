# 加密資產監看

加密資產監看將錢包地址連結到加密資產帳戶，讓您可以檢查餘額與日圓等值金額。從財務報表 -> 加密貨幣進行管理。

除了從正常分錄計算出的餘額之外，應用程式還可以擷取實際的鏈上餘額，並將其反映在財務報表中顯示的加密資產估值中。

## 主要支援型別

| 型別 | 內容 |
| --- | --- |
| BTC | 比特幣地址餘額 |
| ETH | 以太坊地址餘額 |
| SOL | Solana 錢包的 SOL 餘額 |
| SKR | Solana 系列地址的 SKR 餘額 |
| mSOL | 透過 Marinade 質押的 SOL 等值餘額 |
| SOL Stake | Solana 原生質押帳戶餘額 |

幣安整合目前已停用。如果您想管理交易所餘額，請建立加密資產帳戶，並視需要使用手動分錄或實際餘額檢查進行管理。

## 新增錢包

1. 在設定中，建立一個加密貨幣類別的資產帳戶。
2. 開啟財務報表 -> 加密貨幣。
3. 新增錢包，然後選擇鏈、地址及連結的帳戶。
4. 使用餘額擷取檢查可用數量及日圓等值金額。
5. 儲存以在加密資產清單中顯示該錢包。

一般來說，一個帳戶連結一個錢包設定。即使是同一個 Solana 地址，您可能希望分別處理 SOL、SKR、mSOL 等資產，因此在需要時請建立個別帳戶。

## 日圓價值的處理方式

在加密貨幣頁面上，應用程式將擷取的數量乘以價格資料以顯示日圓等值金額。價格從外部價格來源擷取，並可在畫面上重新整理。

對於加密貨幣類別的帳戶，財務報表中顯示的估值可能會被從擷取數量與價格計算出的數值覆蓋，而不是僅從分錄計算出的餘額。這讓您可以更接近當前市值的角度檢視總資產。

## 與分錄的關係

加密資產監看用於檢查持有數量與估值。它不會自動記錄每筆購買、出售、互換、轉帳、手續費、已實現收益或已實現損失。

當您購買加密資產時，仍然需要一筆減少付款資產並增加加密資產帳戶的分錄。當您出售時，記錄現金／存款的增加、加密資產的減少，以及必要時的收益或損失。

## Solana 系列資產注意事項

SOL、SKR、mSOL 及 SOL Stake 可能使用相似的地址格式。自動檢測可能無法正確區分它們，因此在需要時請明確選擇鏈。

Marinade mSOL 與原生質押帳戶使用與一般錢包餘額不同的餘額擷取方法。如果餘額擷取失敗，請檢查地址型別是否正確，以及目標是錢包地址還是質押帳戶地址。

## 管理股票與證券

股票、共同基金等證券透過在資產的投資類別中建立帳戶進行管理。餘額從分錄計算得出。

### 購買分錄

購買股票時使用多行分錄。

| 借方 | 貸方 |
| --- | --- |
| 投資帳戶（持有股數） | 付款帳戶（現金／存款） |
| 購買手續費（支出，如適用） | |

在說明欄位中記下購買價格或成本基礎將有助於日後的損益計算。

### 出售分錄

如果出售價格超過成本基礎，則產生已實現收益；如果低於成本基礎，則產生已實現損失。

**有收益的出售：**

| 借方 | 貸方 |
| --- | --- |
| 收款帳戶（現金／存款） | 投資帳戶（持有股數） |
| | 證券出售收益 |

**有損失的出售：**

| 借方 | 貸方 |
| --- | --- |
| 收款帳戶（現金／存款） | 投資帳戶（持有股數） |
| 證券出售損失 | |

「證券出售收益」與「證券出售損失」可作為系統帳戶使用。

## 記錄加密資產交易

加密資產出售遵循與證券相同的邏輯。

| 型別 | 使用的系統帳戶 |
| --- | --- |
| 出售收益 | 加密資產出售收益 |
| 出售損失 | 加密資產出售損失 |

加密資產監看用於檢查持有量與估值。它不會自動記錄購買、出售、互換或轉帳。請將實際交易另行記錄為多行分錄。

### 估值與帳面價值的差異

當加密資產監看啟用時，財務報表會顯示當前市價的估值。然而，帳面餘額是基於取得時的記錄金額。這兩者之間的差距是未實現損益，在您出售之前不會反映在帳簿中。

對於股票，也沒有自動的每月重估。成本基礎保留在帳簿中，實際損益在出售時記錄。

## 家計財務注意事項

加密貨幣市場價值可能急劇變化。總覽與財務報表中顯示的估值是為了瞭解當前家庭淨值的估算值。它不能完全取代稅務成本基礎、損益計算、移動平均、總平均或手續費處理。

在大額交易後，請另行核實分錄與交易所及錢包記錄是否一致。
