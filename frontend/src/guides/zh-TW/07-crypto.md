# 加密資產餘額核對

餘額核對會將錢包地址連結到加密資產帳戶，讓您比較實際餘額與帳簿餘額。請從財務報表 -> 餘額核對進行管理。

只從鏈上擷取用於餘額核對的持有數量。不擷取市場價值或未實現損益，也不會覆寫財務報表中的帳簿餘額。

## 加密資產的兩種使用模式

帳戶的加密資產類別與貨幣具有不同作用。類別表示持有該資產的目的，貨幣則表示分錄使用的記帳單位。

| 使用模式 | 設定範例 | 主要用途 |
| --- | --- | --- |
| 投資或投機持有 | 在加密資產類別中建立「BTC－投資」帳戶 | 不用於日常消費，將買入、賣出、轉帳、獎勵與手續費記錄為投資活動。可以連結錢包以擷取實際餘額 |
| 日常支付貨幣 | 在貨幣設定中啟用 BTC，並視需要建立「BTC－日常」等現金帳戶 | 透過貨幣切換器選擇 BTC，以 BTC 記錄日常收入、支出與轉帳 |

兩種模式可以同時使用。例如，將「BTC－日常」與「BTC－投資」設為不同帳戶，即可依用途分別管理同一 BTC 貨幣的餘額。餘額核對會按帳戶比較，而不會只根據貨幣代碼將它們合併。

只有在頁首選擇 BTC 等對應加密貨幣時，才會透過明確操作為加密資產類別帳戶擷取錢包餘額。一般頁面重新整理不會擷取餘額。包括日常使用帳戶在內，每個帳戶的實際餘額仍可手動輸入；擷取結果不會覆寫您手動編輯過的值。

## 主要支援型別

| 型別 | 內容 |
| --- | --- |
| BTC | 比特幣地址餘額 |
| ETH | 以太坊地址餘額 |
| SOL | Solana 錢包的 SOL 餘額 |
| SKR | Solana 系列地址的 SKR 餘額 |
| mSOL | 透過 Marinade 質押的 SOL 等值餘額 |
| SOL Stake | Solana 原生質押帳戶餘額 |

幣安整合目前已停用。如果您想管理交易所餘額，請建立加密資產帳戶，並視需要使用手動分錄或實際餘額檢查進行管理。

## 新增錢包

1. 在設定中，建立一個加密貨幣類別的資產帳戶。
2. 在頁首選擇目標加密貨幣，開啟財務報表 -> 餘額核對，然後選擇實際餘額輸入。
3. 新增錢包，然後選擇鏈、地址及連結的帳戶。
4. 使用「擷取並套用」將可用數量填入實際餘額。
5. 儲存快照並檢查它與帳簿餘額之間的差異。

一般來說，一個帳戶連結一個錢包設定。即使是同一個 Solana 地址，您可能希望分別處理 SOL、SKR、mSOL 等資產，因此在需要時請建立個別帳戶。

## 貨幣的處理方式

財務報表會使用可用價格換算帳簿餘額，並以顯示貨幣呈現等值金額。價格來自外部來源，也可以重新整理。

從錢包擷取的數量會保留原始貨幣，只作為餘額核對中的實際餘額；它不會取代分錄計算出的金額。

## 與分錄的關係

餘額核對用於比較持有數量與帳簿記錄。它不會自動記錄每筆購買、出售、互換、轉帳、手續費、已實現收益或已實現損失。

當您購買加密資產時，仍然需要一筆減少付款資產並增加加密資產帳戶的分錄。當您出售時，記錄現金／存款的增加、加密資產的減少，以及必要時的收益或損失。

## Solana 系列資產注意事項

SOL、SKR、mSOL 及 SOL Stake 可能使用相似的地址格式。自動檢測可能無法正確區分它們，因此在需要時請明確選擇鏈。

Marinade mSOL 與原生質押帳戶使用與一般錢包餘額不同的餘額擷取方法。如果餘額擷取失敗，請檢查地址型別是否正確，以及目標是錢包地址還是質押帳戶地址。

## 管理股票與證券

股票、共同基金等證券透過在資產的投資類別中建立帳戶進行管理。餘額從分錄計算得出。

### 購買分錄

購買股票時使用多行分錄。

| 借方 | 貸方 |
| --- | --- |
| 投資帳戶（持有股數） | 付款帳戶（現金／存款） |
| 購買手續費（支出，如適用） | |

在說明欄位中記下購買價格或成本基礎將有助於日後的損益計算。

### 出售分錄

如果出售價格超過成本基礎，則產生已實現收益；如果低於成本基礎，則產生已實現損失。

**有收益的出售：**

| 借方 | 貸方 |
| --- | --- |
| 收款帳戶（現金／存款） | 投資帳戶（持有股數） |
| | 證券出售收益 |

**有損失的出售：**

| 借方 | 貸方 |
| --- | --- |
| 收款帳戶（現金／存款） | 投資帳戶（持有股數） |
| 證券出售損失 | |

「證券出售收益」與「證券出售損失」可作為系統帳戶使用。

## 記錄加密資產交易

加密資產出售遵循與證券相同的邏輯。

| 型別 | 使用的系統帳戶 |
| --- | --- |
| 出售收益 | 加密資產出售收益 |
| 出售損失 | 加密資產出售損失 |

餘額核對用於比較持有量與帳簿記錄。它不會自動記錄購買、出售、互換或轉帳。請將實際交易另行記錄為多行分錄。

### 實際餘額與帳簿餘額的差異

核對畫面會分別顯示擷取的實際餘額，以及依分錄計算的帳簿餘額。如有差異，請檢查錢包歷史並補記遺漏的分錄；擷取餘額本身不會修改帳簿。

對於股票，也沒有自動的每月重估。成本基礎保留在帳簿中，實際損益在出售時記錄。

## 家計財務注意事項

加密資產的市場價格可能急劇變化。總覽與財務報表中顯示的換算值僅用於估算目前家庭淨值，不能取代稅務成本基礎、損益計算、移動平均、總平均或手續費處理。

在大額交易後，請另行核實分錄與交易所及錢包記錄是否一致。
