# 試算表與信用卡對帳

試算表頁面用於檢查帳面餘額是否與實際餘額相符。從財務報表 -> 試算表開啟。

## 功能說明

| 標籤頁 | 用途 |
| --- | --- |
| 實際輸入 | 輸入實際帳戶餘額或信用卡使用金額 |
| 差異驗證 | 將實際值與帳面餘額進行比較 |
| 不明資金 | 暫時吸收無法解釋的差異 |
| 預算一致性檢查 | 檢視分錄與預算之間的不一致及預算配置 |

## 帳戶與信用卡

儲存銀行帳戶、現金、負債或信用卡使用的實際餘額後，差異驗證會顯示帳面價值、實際價值及差異。如果存在差異，請檢查相同金額的候選項、近期分錄、錯誤日期、重複項目、退款及方向錯誤的轉帳。

信用卡經常出現差異，因為使用日、結算日、確認日、扣款日與帳單月份各不同。當帳單確認後，在相關付款月份儲存金額，並與帳簿中的使用金額進行比較。

## 預算一致性檢查

預算一致性檢查驗證分錄的收入／支出金額是否與已儲存的預算分配金額相符。它會檢測缺少分配或金額不符的分錄。

對於多行分錄及預算追蹤範圍外的交易，除非儲存了明確的分配，否則應用程式不會猜測預算類別。如有需要，從帳簿編輯分錄，或使用預算調整僅修正當前預算。

此標籤頁也顯示預算配置表。它將預算類別上設定的目標持有帳戶與實際現金／銀行餘額進行比較，然後顯示關於應在帳戶群組之間移動多少資金或應增加或減少多少預算的提示。
