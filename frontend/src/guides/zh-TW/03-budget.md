# 預算系統（虛擬桶）

預算類別是建立在實際現金與銀行帳戶之上的虛擬桶。實際帳戶餘額保持不動，而預算系統則按用途劃分資金，例如自由花費、必要花費、旅行基金或投資準備金。

## 核心概念

| 項目 | 角色 |
| --- | --- |
| 帳戶 | 實際帳簿項目，例如銀行帳戶、信用卡、薪資或餐飲支出 |
| 預算類別 | 針對現金／存款的虛擬用途桶 |
| 連結的支出帳戶 | 決定哪些預算會被支出分錄消耗 |
| 目標持有帳戶 | 描述您打算將預算資金存放在何處 |

從設定 -> 預算設定建立類別。一個類別可以包含群組、目標餘額、餘額上限、溢位目的地、連結的支出帳戶及目標持有帳戶。

## 結轉、重置與上限

常規預算無論正負均全額結轉。負預算代表超支，因此會減少可分配資金而非增加。

若歷史匯入或修正導致餘額不切實際，請使用輸入 -> 預算調整 -> 預算重置將該類別歸零。總覽頁面會顯示該月的標記日期重置徽章。餘額上限可將超出的預算發送至其他類別。

## 收入分配與調整

收入分錄可套用預算篩選器將存款分配至各類別。篩選器使用固定、上限及比例分割步驟。手動預算調整之後可以增加或減少個別類別，並可包含選擇性備註。

## 預算配置

現金／銀行帳戶具有「納入可分配預算來源」設定。只有啟用的帳戶才會計入可分配現金，並作為預算配置中的實際餘額。

預算配置表出現在設定 -> 預算設定及財務報表 -> 試算表 -> 預算一致性檢查中。

| 顯示 | 含義 |
| --- | --- |
| 目標 | 預期用於該帳戶群組的預算餘額 |
| 實際 | 目標帳戶的實際現金／銀行餘額 |
| 差額 | 實際減目標 |

若一個預算連結到多個帳戶，或多個預算連結到一個帳戶，相關預算與帳戶會合併為一個群組。當存在差額時，提示會顯示應在各帳戶群組之間移動多少資金，或應增加或減少多少預算。

## 轉帳與預算移動

在簡易輸入中輸入帳戶轉帳時，您可以選擇同時移動預算。選擇來源與目的地預算類別以記錄預算轉帳，或將目的地留空以消耗／消失該預算，例如在動用準備金進行投資時。

實際轉帳與預算移動是獨立的事件。預算與帳戶的連結是配置指引；它們不會強制收入分配與銀行轉帳同時發生。

## 預算一致性

試算表 -> 預算一致性檢查可檢測分錄金額與已儲存的預算分配金額之間的不一致。對於多行分錄及預算追蹤範圍外的交易，除非儲存了明確的分配，否則應用程式不會猜測預算類別。
