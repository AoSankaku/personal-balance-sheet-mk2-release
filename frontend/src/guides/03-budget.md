# 予算システム（仮想バケット）

予算カテゴリは勘定科目ではなく、現金・預金の上に重ねる目的別の仮想バケットです。銀行口座の実残高はそのままに、「自由出費」「必要出費」「旅行準備金」「投資準備金」などの使い道別に残高を管理します。

## 基本構造

| 種類 | 役割 |
| --- | --- |
| 勘定科目 | 帳簿に記録される実体。銀行口座、カード、給与、食費など |
| 予算カテゴリ | 現金・預金の使い道を分ける仮想枠 |
| 費用科目との連携 | 支出入力時に、どの予算を消費するかを決める |
| 目標保管口座 | 予算をどの現金・預金口座に置く想定かを示す |

設定 -> 予算設定でカテゴリを作成します。名前、グループ、目標額、上限額、上限超過時の移動先、連携する費用科目、目標保管口座を設定できます。

## 繰越、リセット、上限

通常予算は、プラスでもマイナスでも全額繰り越します。マイナス予算は過去の超過なので、予算可能額を増やすのではなく差し引きます。

過去データの投入や大きな補正で残高が崩れた場合は、入力 -> 予算調整の「予算リセット」で対象カテゴリを0にできます。概要ページでは、リセットがあった月に日付付きバッジが表示されます。

上限額を設定したカテゴリでは、上限を超えた分を別の予算カテゴリへ回せます。

## 収入配分と予算調整

収入入力時は、予算フィルターで入金額を複数カテゴリへ配分できます。フィルターは固定額、上限付き、割合配分のステップで構成します。あとから予算調整で個別に増減でき、調整には任意コメントを残せます。

## 予算配置

現金・預金科目には「予算割り当て元に含める」設定があります。ONの口座だけが、予算可能額と予算配置の実残高に含まれます。

予算配置表は、設定 -> 予算設定と、財務諸表 -> 試算表 -> 予算整合性チェックに表示されます。

| 表示 | 意味 |
| --- | --- |
| 目標額 | その口座グループに置く想定の予算残高 |
| 実残高 | 対象の現金・預金口座の実残高 |
| 差分 | 実残高 - 目標額 |

1つの予算が複数口座に紐づく場合、または複数予算が1つの口座に紐づく場合は、関連する予算と口座をひとつのグループとして合算します。差分がある場合は、どこからどこへいくら移動するか、または追加で割り当てる・減らすべきかのヒントが表示されます。

## 振替と予算移動

シンプル入力で口座間振替を入力するとき、必要に応じて予算も同時に移動できます。送金元カテゴリと送金先カテゴリを選ぶと予算移動として記録され、送金先を空にすると投資実行などで予算が消滅した扱いにできます。

実際の口座振替と予算移動は別イベントです。予算と口座の紐づけは配置確認のための情報であり、収入配分や預金移動を自動で強制するものではありません。

## 予算整合性

支出・収入の仕訳額と保存された予算配分額がずれると、試算表の予算整合性チェックで検出できます。複合仕訳や予算対象外取引には、明示的な配分がない限り予算カテゴリを自動適用しません。
