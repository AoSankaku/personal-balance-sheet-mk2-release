# 複式簿記とシンプル入力

このアプリの帳簿は、すべて複式簿記の仕訳として保存されます。複式簿記では、ひとつの取引を「どこが増えたか」と「どこが減ったか」の両方で記録します。これにより、資産、負債、純資産、収益、費用のつながりを後から検証できます。

ただし、日常の入力で毎回借方・貸方を考える必要はありません。シンプル入力では、取引の種類と金額、相手科目を選ぶだけで、アプリが対応する仕訳を作ります。

## シンプル入力で作られる仕訳

| 入力の種類 | 例 | アプリが内部で行うこと |
| --- | --- | --- |
| 支出 | 銀行口座から食費を支払う | 費用を増やし、支払元の資産を減らす |
| 収入 | 給与が銀行口座に入る | 資産を増やし、収益を増やす |
| 振替 | 普通預金から証券口座へ移す | 片方の資産を減らし、もう片方の資産を増やす |
| 借入 | 友人から立て替えてもらう | 費用または資産を増やし、負債を増やす |
| 返済 | 借りたお金を返す | 負債を減らし、支払元の資産を減らす |
| 貸付 | 誰かにお金を貸す | 貸付資産を増やし、現金・預金を減らす |
| 回収 | 貸したお金が戻る | 現金・預金を増やし、貸付資産を減らす |

## 複式簿記を知らなくても使える理由

シンプル入力では、ユーザーが選ぶのは実生活に近い項目です。たとえば支出なら「何に使ったか」と「どこから払ったか」、収入なら「どんな収入か」と「どこに入ったか」を選びます。借方・貸方の向きは、勘定科目の種類からアプリが判断します。

帳簿ページでは、同じ仕訳を「シンプル」と「複式簿記」の2つの表示で確認できます。慣れていないうちはシンプル表示で内容を確認し、残高がおかしいときや特殊な取引を扱うときだけ複式簿記表示を見る、という使い方で十分です。

## 複合仕訳を使う場面

複合仕訳は、1つの取引に3つ以上の科目が関係する場合に使います。たとえば、手数料を差し引かれて入金された場合、事業立替と個人支出が混在する場合、売却益や売却損を明示したい場合などです。

複合仕訳では、借方合計と貸方合計が一致している必要があります。一致していない取引は保存できません。これは、帳簿全体の整合性を守るための基本ルールです。

## 借方・貸方の基本ルール

複式簿記では、すべての勘定科目が「借方（左）」と「貸方（右）」のどちらかで増え、逆側で減ります。

| 科目の種類 | 増えるとき | 減るとき |
| --- | --- | --- |
| 資産 | 借方 | 貸方 |
| 負債 | 貸方 | 借方 |
| 資本（純資産） | 貸方 | 借方 |
| 収益 | 貸方 | 借方 |
| 費用 | 借方 | 貸方 |

この表を覚えると、仕訳の方向を自分で判断できるようになります。たとえば「銀行口座から食費を払った」という場合、銀行口座は資産なので貸方に記録します。同時に「食費が発生した」なら費用なので借方に記録します。この2行で、1つの仕訳が成立します。

## 仕訳の読み方

帳簿ページの複式簿記表示では、各行が借方または貸方の明細として表示されます。

| 表示 | 意味 |
| --- | --- |
| 借方（左） | その科目が増えた（資産・費用）、または減った（負債・資本・収益） |
| 貸方（右） | その科目が増えた（負債・資本・収益）、または減った（資産・費用） |

一つの仕訳は、必ず借方合計＝貸方合計になります。この均等が崩れている仕訳は保存できません。

## 勘定科目の種類と役割

| 種類 | 例 | 説明 |
| --- | --- | --- |
| 資産 | 銀行口座、現金、貸付金、暗号資産 | 保有する財産 |
| 負債 | クレジットカード、ローン、借入金 | 他者に返す義務がある金額 |
| 資本 | 期首残高 | 純資産（資産から負債を引いた残り） |
| 収益 | 給与、利息収入、売却益 | お金が増える原因 |
| 費用 | 食費、光熱費、減価償却費 | お金が減る原因 |

資本は「資産 - 負債」として計算されるため、直接入力することはほとんどありません。期首残高の入力のみで使います。

## 編集するときの注意

帳簿ページから仕訳を編集できます。シンプル入力で作った仕訳は、可能な範囲で元の入力形式に戻して編集できます。減価償却の月次仕訳や特殊なシステム科目を含む仕訳は、直接編集すると関連する計算が崩れる場合があります。その場合は、元になった入力や設定を変更する方が安全です。

残高が合わないときは、仕訳を推測で消す前に、試算表で実際の残高との差異を確認してください。差異の原因がわかってから修正すると、予算やレポートへの影響も追いやすくなります。
