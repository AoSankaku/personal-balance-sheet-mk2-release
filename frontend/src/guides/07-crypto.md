# 暗号資産ウォッチ

暗号資産ウォッチは、暗号資産用の勘定科目にウォレットアドレスを紐づけ、残高と円換算額を確認するための機能です。財務諸表 -> 暗号資産から管理します。

通常の仕訳から計算される残高だけでなく、ブロックチェーン上の実残高を取得して、財務諸表の暗号資産評価額に反映できます。

## 対応している主な種類

| 種類 | 内容 |
| --- | --- |
| BTC | Bitcoinアドレスの残高 |
| ETH | Ethereumアドレスの残高 |
| SOL | SolanaウォレットのSOL残高 |
| SKR | Solana系アドレス上のSKR残高 |
| mSOL | MarinadeでステーキングされたSOL相当の残高 |
| SOL Stake | Solanaネイティブステークアカウントの残高 |

Binance連携は現在無効化されています。取引所残高を管理したい場合は、暗号資産用の勘定科目を作り、必要に応じて手動の仕訳や現在値確認で管理してください。

## ウォレットを追加する

1. 設定で、暗号資産カテゴリの資産科目を作ります。
2. 財務諸表 -> 暗号資産を開きます。
3. ウォレットを追加し、チェーン、アドレス、紐づける勘定科目を選びます。
4. 残高取得で、取得できる数量と円換算額を確認します。
5. 保存すると、暗号資産一覧に表示されます。

1つの勘定科目には、基本的に1つのウォレット設定を紐づけます。同じSolanaアドレスでも、SOL、SKR、mSOLなどは別の資産として扱いたい場合があるため、必要に応じて勘定科目を分けます。

## 円換算額の扱い

暗号資産ページでは、取得した数量に価格データを掛けて円換算額を表示します。価格は外部の価格情報から取得され、画面上で更新できます。

暗号資産カテゴリの科目については、財務諸表上の評価額が、仕訳だけで計算した残高ではなく、取得した数量と価格から計算される値で上書きされる場合があります。これにより、保有数量の時価に近い形で資産総額を見られます。

## 仕訳との関係

暗号資産ウォッチは、保有数量と評価額を見るための機能です。売買、交換、送金、手数料、売却益、売却損をすべて自動で仕訳化するものではありません。

暗号資産を購入した場合、支払元の資産を減らし、暗号資産科目を増やす仕訳が必要です。売却した場合は、現金・預金の増加、暗号資産の減少、必要に応じて売却益または売却損を記録します。

## Solana系資産の注意

SOL、SKR、mSOL、SOL Stakeは、同じようなアドレス形式を使うことがあります。自動判定だけでは区別できない場合があるため、チェーン選択で明示的に指定してください。

MarinadeのmSOLやネイティブステークアカウントは、通常のウォレット残高とは取得方法が異なります。残高取得に失敗する場合は、アドレス種別が正しいか、対象がウォレットアドレスなのかステークアカウントアドレスなのかを確認してください。

## 株・有価証券の管理

株式や投資信託などの有価証券は、資産カテゴリーの「投資」で勘定科目を作り管理します。残高は仕訳から計算されます。

### 購入時の仕訳

株を購入した場合は、複合仕訳を使います。

| 借方 | 貸方 |
| --- | --- |
| 投資科目（保有株式） | 支払元の口座（現金・預金） |
| 購入手数料（費用、ある場合） | |

購入単価や取得価額は仕訳の説明欄にメモしておくと、後の損益計算に役立ちます。

### 売却時の仕訳

売却価格が取得価額より高い場合は売却益、低い場合は売却損が発生します。

**売却益がある場合：**

| 借方 | 貸方 |
| --- | --- |
| 受取口座（現金・預金） | 投資科目（保有株式） |
| | 有価証券売却益 |

**売却損がある場合：**

| 借方 | 貸方 |
| --- | --- |
| 受取口座（現金・預金） | 投資科目（保有株式） |
| 有価証券売却損 | |

「有価証券売却益」「有価証券売却損」はシステム科目として用意されています。

## 暗号資産の売買記録

暗号資産の売却も、有価証券と同様の考え方で処理します。

| 種類 | 使用するシステム科目 |
| --- | --- |
| 売却益 | 暗号資産売却益 |
| 売却損 | 暗号資産売却損 |

暗号資産ウォッチは保有数量と時価評価を確認するための機能であり、売買や交換を自動的に仕訳するものではありません。実際の売買は別途、複合仕訳として記録してください。

### 評価額と帳簿価額の違い

暗号資産ウォッチを有効にしている場合、財務諸表での評価額は時価ベースで表示されます。一方、帳簿上の残高は取得時の仕訳金額に基づきます。この差は含み益・含み損であり、売却するまでは帳簿に反映しません。

株の場合も同様で、時価に合わせて評価額を毎月自動更新する仕組みはありません。取得価額を帳簿に残し、売却時に実際の損益を記録する方式を採っています。

## 家計管理上の注意

暗号資産の時価は大きく変動します。概要や財務諸表で見える評価額は、家計全体の現在価値を把握するための目安です。税務上の取得価額、損益計算、移動平均、総平均、手数料処理などを完全に代替するものではありません。

大きな売買を行った場合は、帳簿の仕訳と取引所・ウォレットの履歴を別途確認してください。
