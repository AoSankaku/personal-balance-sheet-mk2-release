# 暗号資産の残高照合

暗号資産用の勘定科目にウォレットアドレスを紐づけ、帳簿数量と実際の保有数量を比較できます。財務諸表 -> 残高照合から管理します。

ブロックチェーン上の実残高は照合用の現在値として取得され、財務諸表の帳簿残高を直接上書きしません。差異がある場合は、内容を確認して必要な仕訳を追加します。

## 暗号資産を使う2つのモデル

暗号資産の「勘定科目カテゴリ」と「通貨」は役割が異なります。カテゴリはその資産を何のために保有するかを表し、通貨は仕訳をどの単位で記録するかを表します。

| 利用モデル | 設定例 | 主な使い方 |
| --- | --- | --- |
| 投資・投機として保有 | 資産の「暗号資産」カテゴリに「投機用BTC」科目を作る | 日常決済には使わず、購入、売却、送金、報酬、手数料を投資取引として記録する。ウォレットを紐づけて実残高を取得できる |
| 日常の決済通貨として利用 | 通貨設定でBTCを有効にし、必要なら「日常用BTC」などの現金科目を作る | 通貨切り替えでBTCを選び、日々の収入、支出、振替をBTC単位で記録する |

2つのモデルは併用できます。たとえば同じBTCでも、「日常用BTC」と「投機用BTC」を別の勘定科目にすれば、同じ通貨単位の残高を目的別に管理できます。残高照合も通貨コードだけでまとめず、勘定科目ごとに比較します。

ウォレットからの自動取得は暗号資産カテゴリの科目向けです。日常用科目を含む各科目の実残高は手動でも入力でき、手動で編集した値が自動取得によって上書きされることはありません。

## 対応している主な種類

| 種類 | 内容 |
| --- | --- |
| BTC | Bitcoinアドレスの残高 |
| ETH | Ethereumアドレスの残高 |
| SOL | SolanaウォレットのSOL残高 |
| SKR | Solana系アドレス上のSKR残高 |
| mSOL | MarinadeでステーキングされたSOL相当の残高 |
| SOL Stake | Solanaネイティブステークアカウントの残高 |

Binance連携は現在無効化されています。取引所残高を管理したい場合は、暗号資産用の勘定科目を作り、必要に応じて手動の仕訳や現在値確認で管理してください。

## ウォレットを追加する

1. 設定で、暗号資産カテゴリの資産科目を作ります。
2. 財務諸表 -> 残高照合を開きます。
3. ウォレットを追加し、チェーン、アドレス、紐づける勘定科目を選びます。
4. 「取得して反映」で、ウォレットの実数量を残高照合へ入力します。
5. 保存して差異確認を開き、帳簿数量との違いを確認します。

1つの勘定科目には、基本的に1つのウォレット設定を紐づけます。同じSolanaアドレスでも、SOL、SKR、mSOLなどは別の資産として扱いたい場合があるため、必要に応じて勘定科目を分けます。

## 時価換算と帳簿残高

暗号資産を通貨として有効にすると、帳簿数量は外部の価格情報を使って表示通貨へ換算されます。価格の取得元は外貨設定で管理します。

財務諸表には仕訳から計算した帳簿数量を表示します。ウォレットから取得した実数量は残高照合にのみ使われるため、未記帳の送金や報酬が自動的に財務諸表へ混入することはありません。

## 仕訳との関係

ウォレット連携は保有数量を照合するための機能です。売買、交換、送金、手数料、売却益、売却損を自動で仕訳化するものではありません。

暗号資産を購入した場合、支払元の資産を減らし、暗号資産科目を増やす仕訳が必要です。売却した場合は、現金・預金の増加、暗号資産の減少、必要に応じて売却益または売却損を記録します。

## Solana系資産の注意

SOL、SKR、mSOL、SOL Stakeは、同じようなアドレス形式を使うことがあります。自動判定だけでは区別できない場合があるため、チェーン選択で明示的に指定してください。

MarinadeのmSOLやネイティブステークアカウントは、通常のウォレット残高とは取得方法が異なります。残高取得に失敗する場合は、アドレス種別が正しいか、対象がウォレットアドレスなのかステークアカウントアドレスなのかを確認してください。

## 株・有価証券の管理

株式や投資信託などの有価証券は、資産カテゴリーの「投資」で勘定科目を作り管理します。残高は仕訳から計算されます。

### 購入時の仕訳

株を購入した場合は、複合仕訳を使います。

| 借方 | 貸方 |
| --- | --- |
| 投資科目（保有株式） | 支払元の口座（現金・預金） |
| 購入手数料（費用、ある場合） | |

購入単価や取得価額は仕訳の説明欄にメモしておくと、後の損益計算に役立ちます。

### 売却時の仕訳

売却価格が取得価額より高い場合は売却益、低い場合は売却損が発生します。

**売却益がある場合：**

| 借方 | 貸方 |
| --- | --- |
| 受取口座（現金・預金） | 投資科目（保有株式） |
| | 有価証券売却益 |

**売却損がある場合：**

| 借方 | 貸方 |
| --- | --- |
| 受取口座（現金・預金） | 投資科目（保有株式） |
| 有価証券売却損 | |

「有価証券売却益」「有価証券売却損」はシステム科目として用意されています。

## 暗号資産の売買記録

暗号資産の売却も、有価証券と同様の考え方で処理します。

| 種類 | 使用するシステム科目 |
| --- | --- |
| 売却益 | 暗号資産売却益 |
| 売却損 | 暗号資産売却損 |

暗号資産の残高照合は保有数量の差異を確認する機能であり、売買や交換を自動的に仕訳するものではありません。実際の売買は別途、複合仕訳として記録してください。

### 実残高と帳簿残高の違い

ウォレット実数量と帳簿数量に差がある場合は、未記帳の送金、ステーキング報酬、手数料などを確認します。差異を解消する仕訳を記録するまでは、財務諸表は帳簿数量を基準に表示します。

株の場合も同様で、時価に合わせて評価額を毎月自動更新する仕組みはありません。取得価額を帳簿に残し、売却時に実際の損益を記録する方式を採っています。

## 家計管理上の注意

暗号資産の時価は大きく変動します。表示通貨への換算額は家計全体の現在価値を把握するための目安であり、税務上の取得価額、損益計算、移動平均、総平均、手数料処理などを完全に代替するものではありません。

大きな売買を行った場合は、帳簿の仕訳と取引所・ウォレットの履歴を別途確認してください。
