# 複数通貨の扱い

このアプリでは、複数の通貨を扱うことができます。勘定科目ごとに通貨を設定でき、外貨建て口座や外貨建て資産も帳簿に含められます。

## 基本の考え方

仕訳はすべて通貨ごとに別々に記録されます。たとえば「USD普通預金に1,000ドルが入金された」という取引は、USD通貨として記録されます。

このアプリでは為替レートの自動取得は行いません。外貨建て取引を記録するときは、外貨の金額をそのまま入力します。円に換算した合計値を財務諸表で見たい場合は、別途換算仕訳を入力するか、手動でメモする方法をとります。

## 外貨建て口座を作るには

1. 設定から新しい勘定科目を作ります。
2. 科目の通貨を「USD」「EUR」など目的の通貨に設定します。
3. 初期残高入力で、その時点の外貨残高を記録します。

## 外貨建て取引の記録

外貨建て口座への入出金は、通常通り入力します。金額欄には外貨の金額を入力します。

| 取引の例 | 入力方法 |
| --- | --- |
| USD口座に1,000ドル入金 | 収入として USD口座を選択、金額は1,000 |
| USD口座で100ドルの費用を支払 | 支出として USD口座を選択、金額は100 |
| USD口座からJPY口座に両替 | 複合仕訳で外貨口座と円口座の間で記録 |

## 両替（外貨換算）の仕訳

円建て口座と外貨建て口座の間で換算を行う場合は、複合仕訳を使います。

**例：1,000ドル（取得時レート150円）を145,000円で円に換えた場合**

| 借方 | 貸方 |
| --- | --- |
| 普通預金（JPY）145,000 | USD口座　1,000（取得原価150,000相当） |
| 為替差損　5,000 | |

取得時のレートと両替時のレートの差が為替差益または為替差損になります。この差額を収益または費用の科目として記録します。

このアプリでは、通貨換算仕訳には借方・貸方の金額不一致を許容する設定があります（`is_currency_exchange`）。複合仕訳の入力画面で「通貨換算仕訳」として記録することで、通貨の異なる金額の組み合わせを保存できます。

## 財務諸表での表示

財務諸表の合計欄では、通貨の異なる残高が数値として並びます。たとえば、JPY口座が100,000円、USD口座が500ドルある場合、資産合計には「100,000 + 500」という数値が表示されることになります。

通貨が混在する状態では合計値はあくまで参考として見てください。正確な総資産を把握したい場合は、換算済みの残高を確認するか、手動で換算仕訳を入力して円建てに統一する方法を検討してください。

## 通貨を変更する場合

すでに仕訳がある科目の通貨を変更することはできません。通貨が間違っていた場合は、一括編集・置換機能で仕訳を別の科目に移してから、元の科目を削除してください。

新しい通貨を追加したい場合（例：EUR建て口座を新たに開設）は、EUR通貨の新しい科目を作成します。既存の科目や仕訳には影響しません。

## 暗号資産と通貨

暗号資産（BTC、ETH、SOLなど）も外貨の一種として扱います。暗号資産カテゴリーの科目を作り、通貨を「BTC」などに設定すると、仕訳はその暗号資産の単位で記録されます。

評価額（円換算）は、暗号資産ウォッチを使って外部から価格を取得し、保有数量に掛けることで確認できます。詳しくは「暗号資産ウォッチ」のガイドを参照してください。
