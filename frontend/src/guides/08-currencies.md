# 複数通貨の扱い

このアプリでは、通貨設定で有効にした複数の通貨を切り替えて扱えます。仕訳明細には通貨が記録されるため、外貨や暗号資産の残高を通貨ごとに管理できます。

## 基本の考え方

画面上部の通貨切り替えは、入力・表示の対象となる通貨を選びます。たとえばUSDを選択して「USD普通預金に1,000ドルが入金された」と記録すると、その仕訳はUSDとして保存されます。

勘定科目は1つの通貨に固定されません。同じ勘定科目に複数通貨の残高を持たせることも、通貨や用途ごとに勘定科目を分けることもできます。残高を明確に分別したい場合は、「USD普通預金」「日常用BTC」のような専用科目を作ると分かりやすくなります。

## 外貨を使い始めるには

1. 設定の通貨管理で「USD」「EUR」などを有効にします。
2. 画面上部の通貨切り替えで、使う通貨を選びます。
3. 用途別に残高を分けたい場合は、その通貨専用の勘定科目を作ります。
4. 初期残高入力で、その時点の外貨残高を選択中の通貨として記録します。

## 外貨建て取引の記録

外貨建て口座への入出金は、通常通り入力します。金額欄には外貨の金額を入力します。

| 取引の例 | 入力方法 |
| --- | --- |
| USD口座に1,000ドル入金 | 収入として USD口座を選択、金額は1,000 |
| USD口座で100ドルの費用を支払 | 支出として USD口座を選択、金額は100 |
| USD口座からJPY口座に両替 | 複合仕訳で外貨口座と円口座の間で記録 |

## 両替（外貨換算）の仕訳

円建て口座と外貨建て口座の間で換算を行う場合は、複合仕訳を使います。

**例：1,000ドル（取得時レート150円）を145,000円で円に換えた場合**

| 借方 | 貸方 |
| --- | --- |
| 普通預金（JPY）145,000 | USD口座　1,000（取得原価150,000相当） |
| 為替差損　5,000 | |

取得時のレートと両替時のレートの差が為替差益または為替差損になります。この差額を収益または費用の科目として記録します。

このアプリでは、通貨換算仕訳には借方・貸方の金額不一致を許容する設定があります（`is_currency_exchange`）。複合仕訳の入力画面で「通貨換算仕訳」として記録することで、通貨の異なる金額の組み合わせを保存できます。

## 財務諸表での表示

財務諸表は通常、通貨切り替えで選択した通貨の残高だけを表示します。「全通貨を含む」を有効にすると、通貨設定のレートを使い、ほかの通貨の残高を表示通貨へ換算して合計します。

換算額は価格ソースと取得時点によって変わります。帳簿には元の通貨数量が残り、表示上の換算によって仕訳の数量が書き換わることはありません。

## 通貨を追加・削除する場合

新しい通貨を追加しても、既存の勘定科目や仕訳には影響しません。残高が存在する通貨は無効化できないため、不要になった場合は先に振替や換算仕訳で残高をゼロにしてください。

## 暗号資産と通貨

暗号資産には2つの利用モデルがあります。

- 日常の支払いに使う場合は、通貨設定でBTCなどを有効にし、通貨切り替えから選択して通常の収入・支出・振替を記録します。残高を分けたい場合は「日常用BTC」などの現金科目を作ります。
- 投資・投機として保有する場合は、資産の「暗号資産」カテゴリに「投機用BTC」などの科目を作り、購入・売却・報酬・手数料を投資取引として記録します。

同じBTCを両方の目的で持つこともできます。通貨コードが同じでも勘定科目を分ければ残高は混ざらず、残高照合も科目ごとに行われます。投機用科目はウォレットから実残高を取得でき、日常用を含む各科目は手動でも実残高を入力できます。詳しくは「暗号資産の残高照合」のガイドを参照してください。
